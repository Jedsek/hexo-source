# 空格忽略性
这是一个很简单的性质: macro不匹配传入的空格  
以我们之前玩的 `对应游戏` 为例子:  

```rust
($a: expr, $b: expr) => { $a + $b};    // rule: matcher => transcriber
  |      |  |              |    |
  |      |  |              |    |
( 1      ,  2      ) =>    1 +  2      // pass && expand
```
我们会忽略传入的空格, 以便于在设计 `DSL (自己定义的语法)` 时更加自由, 整齐, 干净
